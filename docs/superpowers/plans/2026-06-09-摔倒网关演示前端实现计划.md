# 摄像头实时摔倒监测演示前端实现计划

> **给 agentic workers 的要求：** 实施本计划时，必须使用 `superpowers:subagent-driven-development`（推荐）或 `superpowers:executing-plans`，按任务逐项执行。步骤使用 checkbox（`- [ ]`）便于跟踪。

**目标：** 构建一个本地只读前端，让导师展示 Fall Edge Gateway 作为“养老照护摄像头实时摔倒监测系统”的项目价值、项目介绍首页、实时监测工作台、告警中心、摔倒事件记录、待复核告警、案例回放和评估指标。

**架构：** 使用 `app.py` 实现一个小型 Python 标准库 HTTP 服务，静态前端放在 `static/`。文件系统数据加载、SQLite 状态合并、摄像头聚合、最高风险状态计算、告警分组、评估汇总和媒体路径安全校验放在 `services/frontend_data.py`，便于单元测试。

**技术栈：** Python 标准库 `http.server`、现有项目服务、SQLite、静态 HTML/CSS/JavaScript、`unittest`。

---

## 范围和约束

- 实现设计文档：`docs/superpowers/specs/2026-06-09-摔倒网关演示前端设计.md`。
- 第一版前端只读。
- 首页是项目介绍页，不是摄像头监控工作台。
- 实时监测工作台作为一个和其他页面同级的导航模块。
- 当前原型使用本地视频模拟摄像头输入，实时监测页摄像头画面用占位图，不假装实时流。
- 网页不能启动 YOLO 或 VLM 任务。
- 不修改 `run_gateway.py`、`run_vlm_worker.py` 或检测流水线行为。
- 数据来源使用现有事件 JSON、`data/records.db`、配置 JSON 和指标 helper。
- 摄像头数量从事件数据里的 `camera_id` 自动生成。
- 事件状态优先使用 SQLite `events.status`，JSON `category` 作为补充。
- 不引入 React、Vue、Vite、Flask、FastAPI 或其他新运行时依赖。
- 保留与本任务无关的现有工作区改动。

## 数据归类规则

前端必须按下面规则归类事件：

```text
confirmed_fall
  -> 实时监测高风险摄像头
  -> 告警中心
  -> 摔倒事件记录
  -> 案例回放与评估可点击案例
  -> 优先使用 SQLite status，JSON category 作为补充

candidates
  -> 实时监测疑似摔倒摄像头
  -> 告警中心
  -> 待复核告警
  -> 案例回放与评估可点击案例

need_human_review
  -> 实时监测待复核摄像头
  -> 告警中心
  -> 待复核告警
  -> 案例回放与评估可点击案例

rejected
  -> 默认不展示事件列表
  -> 不作为可点击案例
  -> 在案例回放与评估中统计数量
  -> 作为“已过滤误报”指标
```

## 摄像头聚合规则

摄像头卡片数量：

- 从事件数据中的不同 `camera_id` 自动生成。
- 第一版不依赖独立摄像头配置文件。
- 没有事件数据时，实时监测页显示“暂无摄像头事件数据”。

摄像头状态优先级：

```text
confirmed_fall > need_human_review > candidates > normal
```

显示文案：

```text
confirmed_fall      -> 已确认摔倒
need_human_review   -> 待复核
candidates          -> 疑似摔倒
normal              -> 正常
```

摄像头画面：

- 第一版显示占位画面。
- 占位文案：

```text
当前为模拟摄像头
暂无实时画面
最近告警可在回放中查看
```


媒体路径使用 URL-safe Base64：

- API 返回的 `media_url` 形如 `/media/<token>`。
- token 编码绝对 clip 路径。
- 服务端解码 token 后，只允许访问配置事件输出目录下的文件。

````markdown
## 前端演示

项目提供一个本地只读前端，用于展示养老照护摄像头实时摔倒监测项目介绍、实时监测工作台、告警中心、摔倒事件记录、待复核告警、案例回放与评估和技术方案。

启动前端：

```powershell
python app.py
```

默认访问地址：

```text
http://127.0.0.1:8000
```

首页是项目介绍页；实时监测工作台作为同级导航模块提供。当前原型使用本地视频模拟摄像头输入，因此实时监测页摄像头卡片使用占位画面，不展示持续实时流。后续接入 RTSP 或真实摄像头后，可以替换为实时视频画面。

前端读取现有的 `data/events`、`data/records.db`、`configs/detection_config.json` 和指标报告数据。第一版前端不会启动 YOLO 或 VLM 推理任务，也不会修改事件数据。

事件分类规则：

- `confirmed_fall` 显示在“摔倒事件记录”，并让对应摄像头显示最高风险状态。
- `candidates` 和 `need_human_review` 显示在“待复核告警”。
- `rejected` 默认不展示事件列表，只在“案例回放与评估”中统计数量。
- 当 `data/records.db` 存在时，前端优先使用 SQLite 中的事件状态。
````
